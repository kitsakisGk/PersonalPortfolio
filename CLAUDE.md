# My Portfolio — Project Rules

## Git Commits
- **Never** add `Co-Authored-By` or any Claude/AI attribution to commits. Every commit is the user's alone.
- Make **multiple small commits** — one per logical change. Never batch everything into one commit.
- Commit messages should be clear, present-tense, and describe what changed (e.g. `Add hero section`, `Update nav styles`).

## Code & Design Standards
- Keep everything on **current trends**: modern stack, modern design patterns.
- Default stack: **Next.js + Tailwind CSS + Framer Motion** unless a specific reason to deviate.
- Design direction: cutting-edge, client-acquisition-focused. Think bento grids, scroll animations, glassmorphism, bold typography.
- No outdated patterns, no jQuery, no Bootstrap.

## Development Approach
- This is a **client-acquisition tool** — every decision should impress potential clients.
- Quality and polish over speed. No half-finished features.
- Test UI in the browser before calling anything done.

## Deployment
- Target: Vercel (primary) or Netlify.
- Every push to `main` should be production-ready.
